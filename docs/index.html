What characterizes a good project?

Simply put, all projects needs the following:

  • Version control: Get away from file data_V1, data_V2, PPT_Final, PPT_FinalV2 etc.
  • Documentation: Clear description of the problem at hand and how to use the programs.
  • Data treatment: Make it clear where data is from and how it is manipulated. Never manipulate an excel spreadsheet manually
  • Modular design: Start with drawing the project, and make a defined split between data treatment and mathematical models.

Most statisticians or data analysts can agree to this in theory. But how to make this easy in real life cases? The next page is meant as a summary of resources and tools to make this easily applicable in all projects. Next comes a small demo case that shows how to use it in practice.
