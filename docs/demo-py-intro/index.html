Single Premium - introduction (work in progress)

Background theory#

In life insurance, reserving isn't based on historical events like in non-life insurance, but is based on a persons income/pension contracts and future life expectency. In matemaical therms:

Tt=SEx+tTt=Value of future premium payoutS=premium intensity based on salary and number of working yearsEx+t=Exptected single premiumT_t = S E_{x+t} \\ T_t = \text{Value of future premium payout}\\ S = \text{premium intensity based on salary and number of working years}\\ E_{x+t} = \text{Exptected single premium}\\Tt​=SEx+t​Tt​=Value of future premium payoutS=premium intensity based on salary and number of working yearsEx+t​=Exptected single premium

The purpose of this model is to calculate the last variable - the exptected single premium, given by the formula

Ex+t=∫0r−yvttpyddtvt=discount factor for time t, given an interest ratetpx=survival probability=Pr(T>x+t)Pr(T>x)where Pr(T>x) is death intensity given by FinanstilsynetE_{x+t} = \int_{0}^{r - y} v^{t} {}_t p^d_y dt \\ v^{t} = \text{discount factor for time t, given an interest rate}\\ {}_t p_x= \text{survival probability} = \frac{Pr(T > x + t)}{Pr(T > x)}\\ \text{where } Pr(T > x) \text{ is death intensity given by Finanstilsynet}Ex+t​=∫0r−y​vtt​pyd​dtvt=discount factor for time t, given an interest ratet​px​=survival probability=Pr(T>x)Pr(T>x+t)​where Pr(T>x) is death intensity given by Finanstilsynet

As stated above, death probability comes from the K2013 tariff made by Finanstilsynet.So the math is quite straight forward, and we have a clear formula that we need to implement. Let's start!

Getting started#

The starting point of all models is to start with drawing. In this calculation a typical inputfile is all pension policies, with promised interest rate, age contract boundries, gender and birthdate. Our output should be a excelreport and an API which other could use. In addition to the model as a package.

graph LR; A[Data] -->|Transform| B[Single Premium]; B --> C{Usage}; C --> D[Excel]; C -->E[API]

After reading the K2013 paper above, we decide to split the functions into two modules, one with the given K2013 functions and one with the actual models. So within the single premium calculation above we get:

classDiagram SinglePremium <|-- K2013_functions class K2013_functions{ death_decline() experience_risk(); } class SinglePremium{ experience_proability() single_premium() }

Now we have the overall structure, a idea of workflow and a clear output. The next step is to create the project before we can start coding.

Get a good starting point by using the project template. This template is made in order to avoid repeating all the boring and time consuming parts for every project, and encourages good programming practice.

From the folder you'll want to create a project type the following into the commandline/shell terminal

cookiecutter https://gitlab.kpmg.no/jalnes/data-science-template

This will ask some questions to initialize the project and tell you to create a git repository. So go to Git, create a repository and copy the git commands from the commandline as seen below. But remember to change REMOTE-repository-url to the link for the repository on GitHub.

With folders created and Git repository created it's time to get started!
