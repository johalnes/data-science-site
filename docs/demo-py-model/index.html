Single Premium - model

Exploratory data analysis#

The next step is to place all given data in the data - raw folder(test data) This is where all input data goes, and should never be directly manipulated. No files from this folder will go into github, both for security and possible large files. Edit the .gitignore files to version controll data.

Jupyter Notebooks are always a nice place to start with testing. Both to get a feel of the data and quickly build and test functions. Either open the folder in VS Code and install the jupyter plugin, or open commandline, cd into project folder and type jupyter notebook.

By opening the example.ipynb from the Notebooks folder, we see that the template comes with the default folder structure, which makes import data easy. Uncomment pandas and numpy, and use pandas to read the excelfile:

From notebook to python script#

Convert notebook#

As stated above, Notebooks are easy and convenient in a exploratory phase, but not as easy to version control and make reusable. For this one should convert to notebook to a regular python script. A good starting point is to convert the notebook to script. Either by running nbconvert command directly !jupyter nbconvert --to script 1-sketch-notebook.ipynb --output ./../src/singlepremium/data_prepp.py

or by choosing export notebook from file menu and rename the file to data_prepp.py within the src - singlepremium folder :

Refactoring#

Typically one start with refactoring and see what typically changes or should be able to parameterize in order to automate the process.

For this demo we split in normal extract, transform and load mindset, even if it's somewhat overkill. Firstly just what we made above:

import os
import sys
import numpy as np
import pandas as pd
from sqlalchemy import create_engine
def extract_data(filepath, filename='policy_data.xlsx'):
df=pd.read_excel(os.path.join(filepath,filename))
return df
def manipulate_data(data):
data=data.drop('Unnamed: 0', axis=1)
df_flat=data.melt(id_vars = df.columns[:3], value_vars=['A6499','A6799'], var_name='Coverage',value_name='Premium')
df_flat['min_age'] = df_flat['Coverage'].str[1:3]
df_flat['max_age'] = df_flat['Coverage'].str[3:5]
return df_flat

Database and pipeline#

Then a regular process usually loads this to a database. For this and connect this to a flow we would need some addidtional code:

#Establishing a database connection, this will be created if it doesn't exist
def create_connection(file_path):
engine = create_engine(f'sqlite:///{file_path}/single_premium_data.db',
connect_args={'detect_types': sqlite3.PARSE_DECLTYPES | sqlite3.PARSE_COLNAMES},
native_datetime=True)
return engine
def insert_in_database(df, filename,engine, calculation_period = (dt.datetime.today() + pd.offsets.MonthEnd(-1)).strftime("%Y-%m")):
#Needed to delete from sql database correctly
df['calculation_period'] = calculation_period
df['filename'] = filename
#Making sure to remove possible duplicates
delete_sql = f"delete from SINGLE_PREMIUM_DATA where filename = '{filename}' and calculation_period = '{calculation_period}'"
engine.connect().execute(delete_sql)
#Appending instead of deliting table, which is safe combined with delete statement above
df.to_sql('SINGLE_PREMIUM_DATA',engine,if_exists='append', index=False)
print('inserted {} rows into SINGLE_PREMIUM_DATA from file:{}'.format(df.shape[0], filename))
if __name__ == "__main__":
project_root = os.path.abspath(os.path.join(os.getcwd(), os.pardir))
data_folder = os.path.join(project_root, 'data')
data_folder_raw = os.path.join(data_folder, 'raw')
data_folder_processed = os.path.join(data_folder, 'processed')
filename='policy_data.xlsx'
engine = create_connection(data_folder_processed)
data = extract_data(data_raw, filename)
data_prepped = manipulate_data(data_prepped)
insert_in_database(data_prepped, filename, engine)
